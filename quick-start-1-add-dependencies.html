--- layout: quick-start group: quick-start sub-group: add-dependencies heading: Add Ebean Dependencies ---

Simply include the Ebean ORM as a dependency in the pom.xml file.

{% highlight xml %} org.avaje.ebeanorm avaje-ebeanorm 3.2.4 {% endhighlight %}

Ebean also requires a logger that implements SLF4J's logger factory. We'll use LogBack classic for this example.

{% highlight xml %} ch.qos.logback logback-classic 0.9.24 {% endhighlight %}

The full pom.xml should now look like this:

{% highlight xml %} 4.0.0 org.avaje.ebeanorm.quickstart ebean-quickstart jar 1.0-SNAPSHOT ebean-quickstart http://maven.apache.org junit junit 3.8.1 test org.avaje.ebeanorm avaje-ebeanorm 3.2.4 ch.qos.logback logback-classic 0.9.24 org.codehaus.mojo exec-maven-plugin 1.2.1 org.avaje.ebeanorm.quickstart.App {% endhighlight %}

We're also going to setup a useful logback configuration file in /src/main/java/resources/logback.xml. Note that we're going to work at "trace" level in order to see the SQL queries generated by Ebean. We're also going to turn off internal Ebean logging to reduce distractions.

{% highlight xml %} %d{HH:mm:ss.SSS} [%thread] %-5level %logger{36} - %msg%n {% endhighlight %}

Let's execute "Hello World!" again.

{% highlight bash %} > mvn install > mvn exec:java ... Hello World! [INFO] ------------------------------------------------------------------------ [INFO] BUILD SUCCESS [INFO] ------------------------------------------------------------------------ {% endhighlight %}
